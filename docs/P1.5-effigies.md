# P1.5 — Effigies & notes with true collected state (implementation notes)

Better than the 1.x approach (per-actor `bPickedInClient`, loaded actors only):
use the game's own global sources, covering streamed-out effigies and old saves.

## Sources (API verified against kit 1.0 headers)

1. **Positions + GUIDs**: skip `UPalLocationManager::GetLocationMap()` (TMap
   via ProcessEvent is painful); instead `FindAllOf("PalLocationPoint")` /
   `"PalLocationPointStatic"` — each location object exposes:
   - `GetLocationId() -> FGuid`
   - `GetType() -> EPalLocationType` (filter Relic / Note values)
   - `GetLocation() -> FVector` (exported, verified present in 1.0)
   Fallback if live enumeration is spotty: static positions from our
   extraction (407 relics / 64 notes in repo Data/), matched by proximity.
2. **Collected flags**: `UPalPlayerRecordData` (via
   `PalUtility::GetLocalRecordData(WorldContext)`, exported) has one
   `RelicObtainForInstanceFlag_<Effect>` per relic bonus type
   (CapturePower, HungerReduction, SwimSpeed, FoodDecayReduction,
   StatusAilmentResist, … enumerate at runtime by property iteration or list
   them explicitly). Each is `FPalPlayerRecordDataRepInfoArrayThreadSafe_BoolVal`
   wrapping `TArray<RepInfo_BoolVal{FGuid Key; bool Value}>`:
   read raw via property pointer + TArray layout {T* Data; int32 Num; int32 Max}.
   Union of all arrays' true-flag GUIDs = collected set.
   Notes: same pattern (`NoteObtainForInstanceFlag`-style — verify exact name
   in PalPlayerRecordData.h when implementing).

## Render

Two new layers (Effigy: green 0x66FF33, Note: cyan 0x33E0FF) using the pooled
dot pipeline; refresh of the collected set on each map open (cheap: a few
hundred GUID hashes into an unordered_set). Optional later: "include
collected" toggles once the P2 panel exists.

## WorldContext for PalUtility statics

Any UObject in-world works (e.g. the map body widget's owning player) — or
call the static with the location point itself as context.
